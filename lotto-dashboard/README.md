# lotto-dashboard

여러 서비스가 동작하는 상황을 한 화면에서 모니터링하기 위해 계획한 대시보드입니다. 현재는 Spring Boot 애플리케이션 골격과 기본 헬스체크 엔드포인트만 존재하며, UI와 실제 연동 로직은 구현되지 않았습니다.

## 현재 상태

- `DashboardApplication`과 기본 Health Controller만 구현되어 있습니다.
- README, 문서상에 소개된 SSE, Web UI, Chart.js, 정적 리소스, 중앙 서버/봇/포스 연동 로직은 전부 (미구현)입니다.
- `application.yml`은 외부 URL을 주입받도록 되어 있지만 실제로 사용되는 빈이나 서비스가 없습니다.

## 앞으로 필요한 작업

1. 정적 리소스(HTML/CSS/JS)와 Controller/Service 계층을 추가해 실시간 데이터를 수집/표시.
2. SSE/REST 클라이언트를 작성해 중앙 서버, 봇 클라이언트, POS 매니저와 통신.
3. 인증/보안, 에러 처리, 테스트 코드 작성.

현재 단계에서는 다른 서비스에서 대시보드로 전송하는 보고 요청이 실패해도 시스템 동작에 영향은 없으며, 로그에만 오류가 출력됩니다.
